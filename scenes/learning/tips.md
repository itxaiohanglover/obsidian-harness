# Learning 场景使用指南

## 推荐插件

| 插件 | 用途 | 安装方式 |
|------|------|---------|
| **Spaced Repetition** | 间隔重复复习闪卡 | 社区插件搜索 "Spaced Repetition" |
| **Templater** | 学习笔记模板自动生成 | 社区插件搜索 "Templater" |
| **Calendar** | 追踪每日学习记录 | 社区插件搜索 "Calendar" |
| **Excalidraw** | 手绘概念图、知识结构图 | 社区插件搜索 "Excalidraw" |
| **Dataview** | 查询学习进度、统计掌握率 | 社区插件搜索 "Dataview" |
| **Web Clipper** | 一键剪存学习资料 | 浏览器扩展商店 |

### Spaced Repetition 配置建议

1. 在笔记中使用 `#flashcard` 标签标记需要复习的卡片
2. 复习间隔推荐：1天 → 3天 → 7天 → 14天 → 30天
3. 配合 `/um flashcard` 自动生成闪卡

## 推荐工作流

### 资料 → 结构化学习库

```
1. 收集资料：Web Clipper / PDF / 手动笔记 → Inbox/
2. /um 学习库 → AI 分析资料结构
                → 识别核心概念
                → 为每个概念创建独立笔记
                → 生成 MOC 仪表盘串联概念
                → 每个主题生成练习题
3. 开始学习！
```

### 互动测验循环

```
/um quiz → AI 推荐测验范围（未测/薄弱区域）
        → 每轮 4 道题，即时反馈
        → 错题换上下文重新出题（不简单重复）
        → 更新进度追踪
        → 一句话总结本次表现
```

### 低摩擦输入

```
读书时：
  手机语音 → "第三章说到 X 概念，我觉得和 Y 有关系..."
  → 坚果云同步到 vault
  → /um → AI 整理为结构化读书笔记

听课时：
  快速记关键词 → 课后 /um → AI 补充展开为完整笔记
```

## 最佳实践

### 学习笔记结构

```markdown
---
mastery: gray
created: 2026-05-19
tags: [learning, subject-name]
last_reviewed: null
---

## 概念名称

### 一句话定义
用最简单的语言说清楚这是什么

### 详细解释
展开说明，包含类比和例子

### 与其他概念的关系
- 前置概念：[[基础概念]]
- 相关概念：[[类似概念]]
- 应用场景：...

### 练习题

> [!question]- Q1: 问题描述
> [!answer]- 答案和解释

> [!question]- Q2: 问题描述
> [!answer]- 答案和解释
```

### 进度追踪颜色约定

| mastery | 含义 | 下一步 |
|---------|------|--------|
| `green` | 已掌握（连续答对 2+ 次）| 7天后复习 |
| `red` | 薄弱（最近答错）| 优先强化 |
| `gray` | 未测试 | 安排诊断测试 |

### 费曼学习法 + AI

```
1. 选择一个概念
2. /um explain → 用自己的话向 AI 解释
3. AI 指出你解释中的漏洞和不准确之处
4. 回到原始资料补充理解
5. 重新解释直到 AI 确认清晰准确
```

### 避免常见陷阱

- ❌ 只收藏不消化 → ✅ 每次收集后立刻 `/um` 结构化
- ❌ 只读不测 → ✅ 每学一个概念就生成练习题
- ❌ 重复刷已会的 → ✅ 间隔重复，只复习薄弱点
- ❌ 笔记越来越多但不关联 → ✅ 每个概念必须链接前置和相关概念
- ❌ 追求完美笔记 → ✅ 先粗记再迭代，AI 帮你补充
