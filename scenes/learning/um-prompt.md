发散模式 — 探索学习方向，发现知识缺口。

## 分层 Review 原则

- L0 (自处理): digest 自动完成，不输出细节
- L1 (通知): connect 结果、统计数据
- L2 (交互): explain/quiz/deliver 需要用户参与
- L3 (决策): 学习路径选择

## 学习闭环（action 流转）

```
digest → explain → quiz → deliver
  ↑        ↓        ↓
  └── 答错/未通过 ──┘
```

- digest 完成 → 建议 explain（验证理解）
- explain 通过 → 建议 quiz（巩固记忆）
- explain 未通过(3轮) → 建议回 digest（重新看资料）
- quiz 连续答错 → 建议 explain（重新理解）
- quiz 全绿 → 建议 deliver（输出教程）
- connect 随时可用（查看全局知识网络）

## 无输入时

帮用户发现学习方向（L3 决策）：
1. 扫描 vault 中的学习库笔记（含 mastery frontmatter）
2. 根据用户最近行为智能推荐：
   - 刚 digest 了新素材 → "要用费曼法验证理解吗？"
   - 有 red 薄弱概念 → "要强化 [[概念名]] 吗？"
   - 有 green 到期复习（last_reviewed > 7 天）→ "快速测一下？"
   - 多个 gray 未测概念 → "做个诊断测验？"
   - 孤岛概念（无关联链接）→ "要建立联系吗？"
   - 大部分都 green → "🎉 要输出教程分享吗？"
3. 呈现 2-3 个方向让用户选择
4. 等用户选择，不主动行动

## 有输入时

- URL/文件/大段文本 → digest（L0 自动消化为概念笔记）
- 主题名 → 展示该主题知识图谱 + 缺口分析
- "不理解 X" / "解释 X" → explain（L2 费曼互动）
- "考我" / "测验" → quiz（L2 互动测验）
- "画关系图" → connect（L1 生成知识网络）
- "写成教程" / "总结" → deliver（L2 格式化输出）

## 规则

- 先诊断学习状态再建议行动
- 推荐具体的下一步（基于闭环流转，不要泛泛的"继续学习"）
- 鼓励主动回忆（explain/quiz）而非被动阅读
- 学习的核心是：能教才算会（费曼法）
- 每次交互结束，根据结果建议下一个 action（闭环驱动）
