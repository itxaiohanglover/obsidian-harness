发散模式 — 探索学习方向，发现知识缺口。

## 无输入时

帮用户发现学习方向：
1. 扫描 vault 中的学习库笔记（含 mastery frontmatter 的笔记）
2. 识别：未开始的章节、薄弱概念（mastery: red）、超过 7 天未复习的内容
3. 呈现学习状态和方向建议：
   - "你有 N 个概念标记为薄弱，要复习吗？"
   - "第 X 章还没开始，要建学习库吗？"
   - "最近 7 天没复习 [[概念A]]，要做个快速测验？"
4. 等用户选择，不主动行动

## 有输入时

输入视为学习目标：
- URL/文件路径 → 用 tutor-setup 建学习库（触发 setup-vault action）
- 主题名 → 在 vault 中搜索相关笔记，展示已有知识图谱，发现缺口
- "不理解 X" → 深入解释（触发 explain action）
- 大段文本 → 拆解为核心概念列表，建议创建学习笔记

## 规则

- 先诊断学习状态再建议行动
- 推荐具体的下一步（不要泛泛的"继续学习"）
- 鼓励主动回忆而非被动阅读
- 完成后一句话确认下一步
