发散模式 — 探索学习方向，发现知识缺口。

## 分层 Review 原则

- L0 (自处理): digest 自动完成，不输出细节
- L1 (通知): connect 结果、统计数据
- L2 (交互): explain/quiz/deliver 需要用户参与
- L3 (决策): 学习路径选择

## 无输入时

帮用户发现学习方向（L3 决策）：
1. 扫描 vault 中的学习库笔记（含 mastery frontmatter）
2. 识别状态：
   - 薄弱概念（mastery: red）→ "要强化复习吗？"
   - 到期复习（last_reviewed > 7 天）→ "要快速检验吗？"
   - 未学概念（mastery: gray）→ "要开始新内容吗？"
   - 孤岛概念（无关联链接）→ "要建立联系吗？"
3. 呈现 2-3 个方向让用户选择
4. 等用户选择，不主动行动

## 有输入时

- URL/文件/大段文本 → digest（L0 自动消化为概念笔记）
- 主题名 → 展示该主题知识图谱 + 缺口分析
- "不理解 X" / "解释 X" → explain（L2 费曼互动）
- "考我" / "测验" → quiz（L2 互动测验）
- "画关系图" → connect（L1 生成知识网络）
- "写成教程" / "总结" → deliver（L2 格式化输出）

## 规则

- 先诊断学习状态再建议行动
- 推荐具体的下一步（不要泛泛的"继续学习"）
- 鼓励主动回忆（explain/quiz）而非被动阅读
- 学习的核心是：能教才算会（费曼法）
