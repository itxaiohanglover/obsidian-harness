# Research 场景使用指南

## 推荐插件

| 插件 | 用途 | 安装方式 |
|------|------|---------|
| **Web Clipper** | 一键剪存网页到 Inbox，保留格式和图片 | 浏览器扩展商店搜索 "Obsidian Web Clipper" |
| **Calendar** | 日历视图查看每日调研笔记 | 社区插件搜索 "Calendar" |
| **Templater** | 调研笔记模板自动填充日期/标签 | 社区插件搜索 "Templater" |
| **Dataview** | 用 SQL 式查询汇总调研进度 | 社区插件搜索 "Dataview" |
| **Excalidraw** | 手绘概念关系图、思维导图 | 社区插件搜索 "Excalidraw" |

### Web Clipper 配置建议

1. 设置默认保存位置为 `Inbox/`
2. 笔记命名格式：`{{date|date:"YYYY-MM-DD"}} - {{title}}`
3. 保管库固定为你的主 vault，避免存错

## 推荐工作流

### 低摩擦收集 → AI 批量消化

```
白天：看到好内容 → Web Clipper 一键保存到 Inbox/
      突发灵感 → 手机备忘录/语音备忘 → 同步到 Inbox/
晚上：/um → AI 自动扫描 Inbox，归档到对应主题笔记
```

### 每日认知简报

```
/aha → AI 读取近 24h 新内容 + 近 7 天笔记
     → 找出 3 个隐藏关联
     → 总结本周核心思考模式
     → 提炼 1 个值得深思的问题
     → 保存为 brief-{日期}.md
```

### 每周深度复盘

```
/aha 复盘 → AI 通读全部调研笔记
          → 识别正在形成的观点
          → 发现新旧认知矛盾
          → 指出知识体系盲区
          → 建议最高杠杆行动
```

## 最佳实践

### 信息卡片原子化

每条信息一个卡片，格式：

```markdown
## 论点标题

- **来源**: [文章名](URL)
- **日期**: 2026-05-19
- **可信度**: 高/中/低

核心内容摘要...

> [!quote] 原文引用
> 关键段落...
```

### 调研主题笔记结构

```markdown
---
status: active
created: 2026-05-19
tags: [research, topic-name]
---

## 核心问题
一句话定义要研究什么

## 子问题
- [ ] 维度 A：...
- [ ] 维度 B：...

## 发现
（信息卡片链接在这里）

## 对比分析
| 维度 | 方案 A | 方案 B |
|------|--------|--------|

## 结论
（综合后的结论，标注来源）

## 信息缺口
（还需要补充什么）
```

### 避免常见陷阱

- ❌ 收集了大量信息但从不回顾 → ✅ 用 `/aha` 定期触发综合
- ❌ 只存不连 → ✅ 每条笔记至少链接 1 个相关笔记 `[[相关主题]]`
- ❌ 追求完美归类 → ✅ 先扔 Inbox，AI 帮你归档
- ❌ 盲目堆积 → ✅ 先拆解问题再收集，带着问题找信息
